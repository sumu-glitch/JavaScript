JavaScript DataTypes

There are two types of datatypes in JavaScript:

1. Primitive

Primitive datatypes are the basic building blocks of data in JavaScript. They include:

  1. String

    Ex: let name = "John";

  2. BigInt

    Ex: let bigNumber = 1234567890n;

  3. Number

    Ex: let a = Number(6);

  4. Boolean

    Ex: let a = true;

  5. Null

    Ex: let a = null;

  6. Undefined

    Ex: let a = undefined;

  7. Symbol

    Ex: let a = Symbol(6);

2. Non-Primitive

Non-primitive datatypes are more complex:

  1. Object

    Ex: let obj = { name: "John", age: 30 };

  2. Array

    Ex: let arr = [1, 2, 3];

  3. Function

    Ex: function greet() { console.log("Hello"); }

  4. Date

    Ex: let currentDate = new Date();

  5. RegExp

    Ex: let regex = /ab+c/;

  6. Error

    Ex: let error = new Error("Oops!");

  7. Map

    Ex: let map = new Map();

  8. Set

    Ex: let set = new Set();

JavaScript is a dynamic language. You don't need to define variable types explicitly like in TypeScript or Java.

Stack() and Heap() Memory

JavaScript uses two types of memory allocation:

  • Stack Memory: Used for static memory allocation. It stores primitive data types and references to objects.

    It's store a perticular element and create that element of copy.

    A deep copy of an object is a copy whose properties do not share the same references

    Ex:
    let a = 5;
    let b = a; // b is now a copy of a
    a = 10; // Changing a does not affect b
    console.log(b); // Output: 5

  • Heap Memory: Used for dynamic memory allocation. It stores Non-Primitive data types like (objects, arrays, and functions).

    in this type of memory it will be give a reference of origional value.

    A shallow copy of an object is a copy whose properties share the same references

    • Ex:
    let obj1 = { name: "John" };
    let obj2 = obj1; // obj2 is a reference to obj1
    obj1.name = "Doe"; // Changing obj1 affects obj2
    console.log(obj2.name); // Output: Doe
